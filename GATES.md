# بروتوكول البوابات (Gates Protocol)

كل ادعاء في بيانات التدريب يمر بهذه البوابات بالترتيب. أي فشل => النص **سام** ويُحجر.

---

## البوابة 0 — أصل التتبّع (Provenance)
- هل المثال يحمل مصدراً موثوقاً؟ (كتاب + صفحة، أو سند متصل، أو مرجع منشور)
- لا مصدر => `unverified` => يُتلف أو يُرجع لمراجعة بشرية.
- **القاعدة:** لا نثق ببيانات بلا سلسلة نقل.

## البوابة 1 — التحقق من القرآن (quran-api / alquran.cloud)
- جلب المصحف كاملاً (114 سورة) حياً.
- تطبيع النص المرشّح: إزالة التشكيل، توحيد الألف/الهمزة/الياء/التاء المربوطة.
- **القبول = مطابقة حرفية تامة** لأية واحدة.
- **الرفض = أي تشابه جزئي أو «قريب منها»** — لا نقبل الاحتمال.
- نص ادّعى أنه قرآن وفشل المطابقة => `POISON_quran_fabricated`.

## البوابة 2 — التحقق من الحديث (dorar-hadith / dorar.net)
- البحث بالنص الكامل.
- التحقق: الراوي + المحدث + المصدر + درجة الصحة.
- **القبول = صحيح أو حسن بسند متصل.**
- ضعيف/موضوع/مقطوع => `POISON_hadith_weak` (لا يُنسب للنبي ﷺ كحكم).
- غير موجود أصلاً => `POISON_hadith_fabricated`.

## البوابة 3 — التحقق من الإجماع (sahaba-ijma / dorar.net)
- توثيق الإجماع بسند متصل.
- غير الموثّق => لا يُرفع لإجماع => `unverified`.

## البوابة 4 — بصمة الذكاء الاصطناعي (AI fingerprint)
- أي نص مولّد بـ AI ثم ادُّعي أنه وحي => سام قطعاً `POISON_ai_forged`.
- علامات: تكرار صيغ، غموض، غياب السند، تناقض مع النص المتواتر.

## البوابة 5 — منع النسبة الخاطئة (No-misattribution)
- كلمة الله ≠ كلمة بشر. لا مساواة.
- أي تسوية بين المصطلح البشري والنص المقدّس => مرفوض.

## البوابة 6 — المراجعة البشرية (Human-in-the-loop)
- الحالات الحدّية => عالم موثوق (أهل الذكر)، لا الذكاء الاصطناعي.

## البوابة 7 — الاختبار العكسي (Red-team)
- إجبار النموذج على «إنتاج آية/حديث» وقياس معدل الهلوسة.
- تجاوز الصفر => الداتاست لا تزال مسمومة.

## بوابة النسبة (Attribution Gate) — القاعدة الذهبية
```
لا ننسب لله ولا لنبيه ﷺ إلا ما ثبت بسند.
كل ما سواه نقول فيه: "لا أعلم".
```

---

## مخرجات الحجر الصحي
كل عنصر مرفوض يُسجَّل بـ:
- `claim_type` (quran/hadith/ijma/ai_forged)
- `reason` (سبب الرفض)
- `confidence` (درجة اليقين في الرفض)
- `source_checked` (أي API رُاجع)
