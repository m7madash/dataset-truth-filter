# Dataset Truth Filter — مُرشّح حقيقة البيانات

أداة مفتوحة المصدر للكشف عن **تسميم بيانات التدريب (data poisoning)** في مجموعات بيانات النماذج اللغوية، خصوصاً الحالة الأخطر:

> نسبة كلام بشري — أو نص مُولّد بالذكاء الاصطناعي ثم عُدّل — إلى أنه **قرآن أو حديث أو وحي**، وتقديمه للنموذج على أنه نص مقدّس.

هذا ليس خطأ تقنياً فحسب — إنه **تزوير في النسبة**، ويُنتج نماذج تكذب على الناس باسم الدين.

---

## المبدأ (غير قابل للتفاوض)

```
كلمة الله لا تساوى كلمة بشر.
لا ننسب لله ولا لنبيه ﷺ إلا ما ثبت بسند متصل.
كل ما سواه => "لا أعلم" (غير موثّق).
```

لذلك المستودع **لا يخزّن أي نص مقدّس**. كل التحقق يجري **حيّاً (live)** عبر واجهات برمجية موثوقة:
- `alquran.cloud` — مصحف المدينة (للتحقق من الآيات)
- `dorar.net` (الدرر السنية) — للتحقق من الأحاديث والآثار والإجماع

النص المقدّس يُجلب للتحقق فقط، ولا يُحفظ في هذا الـ repo.

## كيف يعمل (خط الأنابيب)

1. **البوابة 0 — أصل التتبّع (Provenance):** كل مثال تدريبي يجب أن يحمل مصدراً (كتاب/صفحة/سند). من بلا مصدر => يُتلف أو يُوسم `unverified`.
2. **بوابة القرآن:** مطابقة **حرفية مُطبّعة** (تطبيع التشكيل والألف/الهمزة) ضد المصحف كاملاً. القبول = المطابقة التامة فقط. لا تشابه لغوي (cosine) — التشابه = احتمال، والموضوع لا يحتمل الاحتمال.
3. **بوابة الحديث:** التحقق عبر الدرر السنية — النص الكامل + الراوي + المصدر + درجة الصحة. القبول = صحيح/حسن بسند.
4. **بوابة الإجماع:** توثيق الإجماع بسند متصل.
5. **بوابة الذكاء الاصطناعي:** أي نص مولّد بـ AI ثم ادُّعي أنه وحي => سام قطعاً.
6. **الحجر الصحي (Quarantine):** كل ادعاء فشل => يُنقل لملف منفصل مع سبب الرفض، لا يُدخل التدريب.

## الاستخدام

```bash
# التحقق من ادعاء واحد
python3 filter/verify_claim.py "نص يُدّعى أنه قرآن أو حديث"

# تنقية داتاست كامل (JSONL أو JSON)
python3 filter/purify_dataset.py path/to/dataset.jsonl --out clean.jsonl --quarantine poison.jsonl
```

مخرجات:
- `clean.jsonl` — بيانات نظيفة اجتازت البوابات
- `poison.jsonl` — بيانات سامة مرفوضة مع سبب لكل عنصر
- تقرير على stdout: نسبة السمّ، أنواعه، توصية

## حدود الأمانة

- هذه أداة **هندسة بيانات وضوابط**، ليست مُفتياً. لا تصدر حكماً شرعياً مُلزماً.
- الحالات الحدّية ترجع لـ **أهل القرآن وبيان النبي ﷺ** (علماء موثوقون) — لا يحسمها الذكاء الاصطناعي.
- المطابقة تتطلب اتصالاً بالإنترنت (التحقق حيّ). بدون اتصال => كل الادعاءات `unverified`.

## المساهمة

نرحّب بكل مؤسسة أكاديمية أو جهة شرعية أو فريق أمان ذكاء اصطناعي تريد:
- تبني الفكرة ودمجها في خطوط تدريبها
- تحسين بوابات التحقق
- إضافة مصادر موثوقة أخرى

انظر `GATES.md` للبروتوكول الكامل.
